Ajax学习笔记

Ajax概述

Ajax：它是浏览器提供的一套方法你可以实现页面无刷新更新数据，提高用 户浏览网站应用的体验

Ajax应用场景

  1. 页面上拉加载更多数据
  2. 列表数据无刷新分页
  3. 表单离开焦点数据验证
  4. 搜索框提示文字下拉列表

Ajax运行环境

Ajax技术需要运行在网站环境中才能生效。

Ajax运行原理

Ajax相当于浏览器发送请求与接收响应的代理人，以现在不影响用户浏览页面的情况下，局部更新页面数据，从而提高用户体验。

Ajax实现步骤

  1. 创建Ajax对象(请求代理人)
  2. 告诉Ajax请求地址以及请求方式
  3. 发送请求
  4. 获取服务器端给予客户端的响应数据

    服务器端响应的数据格式

    在真实的项目中，服务器大多数情况下会以JSON对象作为响应数据的格式，当客户端拿到数据时，要以JSON数据和HTML字符串进行拼接，然后将拼接的结果展示在页面中。

在http请求与响应的过程中，无论是请求还是响应内容，如果是对象类型，最终都会被转换为对象字符串进行传输

请求参数传递

  • 传统网站请求
  • GET请求方式 ==注意发送GET请求时请求参数自己拼接==
  • POST请求 ==post请求学习时遇到的问题：==
    node.js提示: req.body为undefined：原来是在较新版本中的express组件中已经分离了body-parser这个中间件，如果没有默认依赖，需要手动添加

    请求参数的格式(post来说)

  • application/x-www-form-urlencoded
  • appiccation/json 在请求头中指定Content-Type属性的值是application/json，告诉服务器端当前的请求参数的格式是json ==注意：==
    get请求是不能提交json对象数据格式的，传统网站的表单提交也是不支持json对象数据格式的。

    获取服务器端的响应（方法二）

    Ajax状态码

    在创建ajax对象，配置ajax对象发送请求，以及接收完服务器端的响应数据，这个过程中的每一个步骤都会对应一个数值，这个数值就是ajax状态码。
  • 0请求还未初始化（还未调用open()）
  • 1请求已经建立，但是还没有发送（还没调用send()）
  • 2请求以发送
  • 3请求处理中，通常响应中已经有部分数据可以用了
  • 4请求已经完成，可以获取并使用服务器端的响应了 在请求发送的过程中状态码是不断变化的，所以在send()方法后调用xhr.readyState无法看到效果所以为了看到这些状态码；onreadystatechange函数绑定在send()前通过判断状态码，然后对响应数据进行操作

    两种获取服务器端响应方式的对比

    Ajax错误处理

  • 网络畅通，服务器端能接收到请求，服务器端返回的结果不是预期结果(400 (Bad Request))
    可以判断服务器端返回的状态码，分别进行处理。xhr.status获取http状态码
  • 网络畅通，服务器端没有接收到请求，返回状态404状态码(404 (Not Found))
    检查请求地址是否错误
  • 网络畅通，服务器端能接收到请求，服务器端返回500状态码(500 (Internal Server Error))
    服务器端错误，找后端程序员进行沟通
  • 网络中断，请求无法发送到服务器端
    不会触发onload事件，会触发xhr对象下的onerror事件，在onerror事件处理函数中对错误进
    行处理。

    Ajax状态码与http状态码

    Ajax状态码：表示Ajax请求的处理过状态， ajax对象返回的
    http状态码：表示请求的处理结果 是服务器端返回的

    部分http状态码的含义：

  • 200 OK
  • 400 Bad Request
  • 404 Not Found
  • 500 Internal Server Error、

    低版本IE浏览器的缓存问题(我测试IE8的时候没有问题)

    问题：在低版本的IE浏览器中，Ajax请求有严重的缓存问题，即在请 求地址不变的情况下只有第一次请求会真正发送到服务器后端，后续的请求都会从浏览器的缓存中获取结果。即使服务器端的数据更新了，客户端依然拿到的是缓存中的旧数据。
    解决方案：在请求地址后面添加请求参数，保证每一次请求中的请求参数的值不同。

    同步异步概述

    同步

    一个人同一时间只能做一件事，只有一件事做完才能做另一件事
    对于代码来说，就是上一行代码完成，才能执行下一行代码，即逐行执行

    异步

    一个人一件事情做了一半，转而去做其他事情，当其他事情完成后，再回头来继续完成之前未完成的事。
    对于代码来说，就是异步代码虽然需要花时间去执行，但程序不会等异步代码结束再继续执
    后续代码，而是直接执行后续代码，当后续代码执行结束之后，再回头看异步代码是否返回
    结果，如果已经返回结果，再调用事先准备好的回调函数处理异步代码执行结果。

    Ajax封装

    问题：发送一次请求代码过多，多次请求代码冗余复杂。
    解决方案：将请求代码封装到函数中，发送请求时调用即可 调用(封装时设置了默认参数，所以有些参数可以不用传递)
